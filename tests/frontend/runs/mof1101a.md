2. **Gemini Vertex wiring required three fixes to actually work**:
   - Mount the SA keyfile into the bot container — `docker-compose.yml`
     gained `/home/mulligan/alexa2.json:/secrets/alexa2.json:ro`.
   - Update `.env` `GOOGLE_APPLICATION_CREDENTIALS` to the in-container
     path (`/secrets/alexa2.json`) so the SDK can read it.
   - Patch `src/agent/handlers.js:334` — the `generate_image` handler
     was guarding only on `config.gemini.apiKey`, so even with Vertex
     env vars set the handler short-circuited before the Gemini client
     ever saw the Vertex branch. Now guards on
     `!apiKey && !vertex.project`.
   - Grant `roles/aiplatform.user` to the SA on project `alexa2-ea8b4`
     (one-time IAM bind, took ~60s to propagate).

   After all of the above, `gemini-2.5-flash-image` works on Vertex.
   T5.4 (chat-context only, no explicit prompt) returns empty image
   data — the model needs at least a thin prompt fragment to anchor on;
   passing an explicit prompt + `include_recent_chat=true` works.

3. **`tavily_show_image` now refuses URLs not from a recent
   `tavily_search`** — T7.3 fails because the bot enforces this
   constraint. The catalog feeds it `URL_A` (a fresh Wikimedia URL), so
   the call is rejected. Either:
   - Update the catalog T7.3 prompt to first run a Tavily search and pick
     a URL from the result, OR
   - Loosen the `tavily_show_image` handler to accept any HTTP(S) image
     URL (matching `add_beat_image` / `add_character_image`), since the
     image-bytes validator already enforces type/size guards.

4. **PDF export still falls back to localhost URL** (same as last run) —
   T8.1 + T8.2 generate the PDF (verified end-to-end via T9.2 — 2.87 MB
   `application/pdf` served by the web server) but never produce a
   Discord file attachment. Discord's 25 MB cap is well above 2.87 MB, so
   it's not a size issue; worth investigating whether the
   `__PDF_PATH__:` interceptor / `sendReply` attach path is broken for
   this code path or if the LLM is choosing the URL approach over the
   sentinel.
